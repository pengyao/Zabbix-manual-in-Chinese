关于导出的信息
-----------------

    * 所有支持的元素都能导出到一个文件中
    * 从模板继承的主机和模板元素,不会被导出.
    * LLD创建的元素和任何依赖于它们的元素将不会被导出.例如,如果创建了一个针对LLD规则生成的项目, 这样的触发器不能被导出
    * 用于web项目的触发器和图形不能导出
    * 在前端,配置中能被导出成XML格式,使用API的方式XML和JSON都支持
    
.. warning::
      
        在2.0.1之前处理不同的空标签语法不相同

关于导入的信息
----------------
    * 在遇到第一个错误时导入停止
    * 当在导入图片是更新一个已存在的图片,"imagetype"字段将被忽略,也就是说,不能够在导入时改变图片类型.
    * 空的项目,触发器,图形,主机/模板应用,发现规则,项目原型,触发器原型,图形原型标签是没有意义的,也就是说等同于漏掉它.其它的标签,例如项目应用是有意义的,空标签意味着项目中没有应用,漏掉标签意味着不更新应用
    * 导入对XML和JSON都支持,导入的文件的扩展名必选正确: .xml对应XML .json对应着JSON.
    
XML基础格式
--------------
::

    <?xml version="1.0" encoding="UTF-8"?>
    <zabbix_export>
        <version>2.0</version>
        <date>2012-04-18T14:07:36Z</date>
    </zabbix_export>

::

    <?xml version="1.0" encoding="UTF-8"?>

XML文档的默认头部

::

    <zabbix_export>

Zabbix XML导出的根元素

::

    <version>2.0</version>

导入版本

::

    <date>2012-04-18T14:07:36Z</date>

在导入的创建时间使用ISO 8601长格式.
其他标签是依赖于导出的对象。
